DevTools Masterclass

Follow the steps below with your Console open!

1. The Elements Tab (The Skeleton)

The "Elements" tab shows the DOM (Document Object Model). It's the live version of your HTML.

Action: Right-click the button below and select "Inspect". Try changing the text color of this button in the "Styles" pane on the right!

2. The Console (The Brain)

The "Console" is where we talk to the computer. You can see logs and run code live.

Action: Click the button to send a message to the console. Notice the blue file name (e.g., index.html:105)—that tells you exactly which line sent the message!

3. Reading Errors (The Red Alerts)

Errors aren't failures; they are roadmaps and descriptors. They tell you What went wrong and Where.

Action: Click the button below. It will trigger a "ReferenceError". Look for the red text in the console and click the link on the right to see the broken line.

4. The Debugger (The Time Machine)

The "Sources" tab lets you pause time using Breakpoints.

Action: Click the button. The code will pause mid-execution. You can then hover over the variable secretMessage in your DevTools to see its value!

5. The Network Tab (The Fetcher)

The "Network" tab shows every file being downloaded (images, scripts, data).

Action: Open the Network tab, then click the button. You will see kitty.webp pop up in the list as it downloads!
